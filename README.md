# JerichoStatus

Офлайн Android-приложение для учета игровых статусов, эффектов и инвентаря в ролевой страйкбольной игре по постапокалиптической вселенной.

## Возможности

- быстрый запуск игровых статусов из пресетов;
- автоматический переход по цепочке ранений по таймеру;
- блокировка запуска таймеров иммунитетами и защитами;
- активные таймеры с паузой, ручным завершением и лечением;
- предложение сразу применить предмет при старте болезни, если лечение доступно;
- отдельные вкладки для постоянных эффектов, истории, инвентаря и профиля игрока;
- инвентарь с добавлением, расходом, передачей по QR и приемом через камеру;
- журнал истории по статусам и инвентарю;
- фиксация источника получения предметов;
- хранение всех данных локально на устройстве без сервера.

## Основные экраны

- `Статусы`:
  запуск пресетов, запуск собственного таймера, подсказка о мгновенном лечении.
- `Активные`:
  текущие таймеры, пауза, завершение, лечение, отображение прогресса.
- `История`:
  завершенные статусы, переходы между стадиями, получение и расход инвентаря.
- `Постоянные`:
  постоянные иммунитеты и защиты.
- `Инвентарь`:
  список предметов, пополнение, расход, добавление с указанием источника, QR-передача и QR-получение.
- `Профиль игрока`:
  игровые данные бойца и реальные данные для экстренных случаев.

## Игровая логика, которая уже реализована

- Дубликаты одинаковых таймеров не запускаются.
- Пока активен `Мертвяк`, новые таймеры навешивать нельзя.
- При входе в `Мертвяк` снимаются болезни, ранения и положительные эффекты с записью в историю.
- Болезни можно лечить подходящими предметами:
  - `Серая гниль` -> `Антибиотик` или `Зеленое зелье`
  - `Пляска смерти` -> `Синее зелье`
  - `Морок` -> `Фиолетовое зелье`
- При старте болезни приложение может сразу предложить применить лечение, если предмет уже есть в инвентаре.
- Для ранений работает автопереход:
  `Легкое ранение -> Тяжелое ранение -> Критическое состояние -> Смерть: ожидание на месте -> Мертвяк`

## Инвентарь

Поддерживаются:

- зелья;
- антибиотики;
- категории чипов;
- лог получения, расхода, передачи и приема по QR;
- добавление предмета с источником:
  - `Найден в открытом мире`
  - `Добыт в бою`
  - `Получен от соратника`
  - `Получен по бартеру`

Для источников `Добыт в бою`, `Получен от соратника` и `Получен по бартеру` сохраняется дополнительная информация по позывному/контакту.

## QR-передача

- На экране передачи выбирается предмет и количество.
- Перед показом QR приложение запрашивает подтверждение.
- После подтверждения предмет сразу списывается у отправителя и пишется в историю.
- Получатель сканирует QR камерой, предмет добавляется в инвентарь и тоже пишется в историю.
- Повторный прием одного и того же QR на одном устройстве блокируется.

- `RecyclerView`

- `ZXing` / `zxing-android-embedded` для QR

- `minSdk 26` (`Android 8.0`)
- `targetSdk 35`
- `compileSdk 35`

```bash
./gradlew :app:assembleDebug
```

```text
app/build/outputs/apk/debug/app-debug.apk
```

## Разрешения

- `POST_NOTIFICATIONS`:
  уведомления о завершении таймеров.
- `CAMERA`:
  сканирование QR-кодов для приема предметов.

## Хранение данных

Все данные сохраняются локально в `SharedPreferences`:

- активные статусы;
- история статусов;
- постоянные эффекты;
- инвентарь;
- история инвентаря;
- уже принятые QR-передачи;
- профиль игрока.

Приложение не требует сети для основной работы.

## Структура проекта

- `app/src/main/java/com/masolodilov/jericho/data`
  репозиторий, каталог пресетов, ticker, storage, QR codec.
- `app/src/main/java/com/masolodilov/jericho/model`
  модели статусов, инвентаря, профиля и результатов операций.
- `app/src/main/java/com/masolodilov/jericho/ui`
  активити, фрагменты, адаптеры, уведомления, форматирование времени.
- `app/src/main/res/layout`
  XML-экраны и карточки интерфейса.
